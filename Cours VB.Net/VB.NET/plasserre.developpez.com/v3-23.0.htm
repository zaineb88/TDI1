3.23.0 Applications et formulaires WPF.

 

Les contrôles WPF font partie de l'espace de nom System.Windows.Controls (ne pas confondre avec System.Windows.Forms des contrôles Winforms habituels)  

 

Structure d'une application:

Créons une application WPF nommée MonApplicationWPF dans VB. Un formulaire Window1 est crée. On y ajoute un module  et une Classe.

L'explorateur de solution en haut à droite permet de voir de quoi se compose le projet.

MyProjet donne accès aux propriétés du projet.

Application.xaml est le fichier d'application:

<Application x:Class="Application"

xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"

xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"

StartupUri="Window1.xaml">

<Application.Resources>

 

</Application.Resources>

</Application>

 

Si l'application a été créée dans Blend, on peut voir dans vb que cela diffère un peu:

xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"

Dans un formulaire, il y a toujours en conteneur parent, ici c'est <Grid>. Si l'application avait été créée dans Blend cela serait <FlowPanel>.

Ouvrir une fenêtre:

Ajouter une fenêtre au projet:

Menu 'Projet', puis 'Ajouter une fenêtre'.

Cliquer sur Fenêtre WPF puis sur le bouton 'Ajouter'.

On a crée une seconde fenêtre nommée Window2

Dans le code vb de la première fenêtre, comment ouvrir Window2? 

Dim w As New Window2

w.Show()

ou

w.ShowDialog() pour ouvrir une fenêtre modale.
